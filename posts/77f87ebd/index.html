阿里云oss+PicGo实现图片自动上传到图床

阿里云OSS+PicGo实现图片自动上传到图床

1.阿里云oss（nas也可，做nginx反向代理，按个人情况选择即可）

注册阿里云，创建oss，桶权限必须设为公有读私有写+开启防盗链，防止别人盗刷流量（必开），否则网页打开图片 403 无权限，创建api密钥。详细操作步骤请自行查找相关资料。

注意：创建完RAM用户，给它授予 AliyunOSSFullAccess 权限，拿到两组密钥，就可以填进 下列安装的PicGo 了

✅ 对比本地图片两大优势：

  1. 永不裂图：Hexo 部署到服务器 / GitHub Pages ，不受目录结构影响，永远不会 404 裂图
  2. 减轻服务器压力：图片全部走阿里云 CDN 加速，不走你的 ECS 服务器带宽，打开网页速度更快，不用跟着静态文件一起打包上传。

2.安装配置PicGo

图床管理工具我选了picgo

  1. GitHub 原版（稳定版）

    https://github.com/Molunerfinn/PicGo/releases

    找到最新版，下载

    国内快速下载（打不开 GitHub 就用这个）

    https://picgo.app/zh/download/

  2. PicList安装步骤

桌面安装即可。

  1. 安装完成后，桌面会出现 PicGo 图标，打开软件。

  2. 接下来配置阿里云 OSS

  3. 打开 PicGo → 左侧【图床设置】→ 选择【阿里云 OSS】

  4. 把 RAM 子账号的密钥、Bucket 名称、区域填进去：

    • KeyID：AccessKey ID

    • KeySecret：AccessKey Secret

    • Bucket：你创建的桶名

    • 存储区域：oss-cn-hangzhou

    • 自定义存储路径：blog/

  5. 控制台里还有一种带时效的临时签名 URL，不要用这种，过期会打不开图片

  6. 图床勾选只保留一个

    ​ 只勾选 阿里云 OSS，把腾讯云、shtur、GitHub 等其他全部取消勾选，界面干净，不会选错图床。

    ​ 回到左侧【图床设置】，确认当前默认图床为阿里云 OSS，就可以拖拽图片测试上传了

  7. PicGo 上传成功后会自动把链接复制到剪贴板，不需要反复进 OSS 控制台复制。

  8. 控制台里还有一种带时效的临时签名 URL，不要用这种，过期会打不开图片

3.上传效果

只要你粘贴截图、插入本地图片，Typora 会自动把图片丢给 PicGo，自动上传 OSS，自动把图片替换成 OSS 外网链接。

4.执行结果

  1. 电脑截图 → PicGo 自动上传 OSS → 复制外链
  2. 远程连接阿里云 ECS，打开 md 文档，粘贴图片链接
  3. 写完执行
  1. 博客自动更新，图片全部加载正常。
